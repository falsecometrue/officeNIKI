# UT_F02 Word→Markdown変換機能 テスト仕様書

## 1. 目的

Word→Markdown変換機能について、Word文書の主要要素、画像外部管理、Markdown出力、フォルダ一括変換、処理時間の観点で確認する。
変換結果は、AIが文書内容を理解しやすい構造を優先し、次に人間が視認、編集しやすいMarkdownであることを確認する。

## 2. テスト方針

- 変換対象は `.docx` とする。
- 出力は入力ファイルと同一フォルダに作成した入力Wordファイル名フォルダ内のMarkdownとする。
- 中間JSONファイルは出力しない。
- 画像はbase64ではなく、`<入力Wordファイル名>resource/` フォルダ配下へ外部ファイルとして出力する。
- Markdown本文から画像は相対パスで参照する。
- 文字色、文字サイズ、配置などのテキストデザインはHTML混在Markdownとして出力してよい。
- チャートは画像化より先にMermaid変換を試み、Mermaid化できない場合はデータ表として出力する。
- HTML tableによる左右レイアウトは採用しない。
- Word表示との完全一致は判定対象外とし、本仕様書に記載したMarkdown記法、出力ファイル数、画像ファイル数、リンク数、処理時間を合格基準とする。

## 3. テスト観点

| 観点 | 確認内容 | 期待結果 |
|---|---|---|
| 見出し | Title、Heading1、Heading2、Heading3 | 対象段落が `#`、`##`、`###`、`####` のMarkdown見出しとして出力される |
| 段落 | 通常段落、複数段落 | 段落テキストが本文として出力され、段落間に空行が1行以上入る |
| テキストデザイン | 太字、斜体、下線、文字色、文字サイズ、中央揃え、右揃え | Markdown強調、`span style`、`u`、`p style` のいずれかで装飾情報が出力される |
| 改行 | 段落内改行、表セル内改行 | 段落内改行はMarkdownで読める改行として出力され、表セル内改行は `<br>` として出力される |
| 箇条書き | 箇条書き、番号付きリスト | 箇条書きは `- `、番号付きリストは `1. ` で出力される |
| 表 | Word表、列数違いの表 | Markdown tableとして出力され、ヘッダー区切り行 `|---` が1件以上出力される |
| 画像 | Word文書内画像 | 入力画像数と同数の画像ファイルが `<入力Wordファイル名>resource/` に出力され、Markdown内に同数の `![](` 画像リンクが出力される |
| 画像プレースホルダー | 1x1画像 | 1x1画像はresourceフォルダに出力されず、Markdown画像リンクも出力されない |
| Word図形 | 図形、テキストボックス、単純フロー図 | 単純フロー図はMermaidコードブロック、テキストのみ取得できる図形は本文テキスト、画像化のみ可能な図形は画像リンクとして出力される |
| Mermaid候補 | 単純な開始・処理・終了のフロー図 | Markdown内に ` ```mermaid ` と `flowchart` が1件以上出力される |
| チャートMermaid | 棒、折れ線、円などの単純チャート | Markdown内に ` ```mermaid ` とチャート内容を表すMermaidコードが出力される |
| チャートデータ表 | Mermaid化できないがデータ抽出できるチャート | チャート名、系列、カテゴリ、値を含むMarkdown tableが出力される |
| HTML tableレイアウト | 画像横に説明文があるWordレイアウト | Markdown内にレイアウト目的の `<table>` が出力されず、画像リンクと本文段落として出力される |
| 出力先 | 右クリック選択したWordファイル | 入力Wordと同一フォルダに入力Wordファイル名フォルダが作成され、その配下に `.md` とresourceフォルダが出力される |
| 複数ページ | 1〜100ページ相当の文書 | 本文順が維持され、先頭ブロックから末尾ブロックまでMarkdownに出力される |
| フォルダ一括 | 1〜100ファイル | 入力Word件数と同数のMarkdownが各入力ファイル名フォルダに出力される |
| 処理時間 | 通常サイズの単一ファイル | 3秒以内にMarkdown生成が完了する |
| エラー | Officeとして開けないファイル、パスワード付きファイル | Markdownとresourceを出力せず、エラー終了する |
| 警告 | 画像参照切れ、未対応図形 | 変換可能な要素を含むMarkdownを出力し、対象要素は警告として扱う |

## 5. 性能目標

| 項目 | 目標 |
|---|---|
| 通常サイズの単一Word | 3秒以内 |
| 文書規模 | 100ページ相当まで対応 |
| 画像数 | 100画像まで対応 |
| 表数 | 100表まで対応 |
| フォルダ一括 | 100ファイルまで対応 |

3秒目標は通常サイズの単一ファイルを対象とする。
100ページ、100画像、100表、100ファイルのケースでは、3秒を超えるケースとして処理完了可否とログ出力を確認する。

## 6. 合格基準

- 正常系の全ケースでMarkdownが出力され、UTF-8で読み込めること。
- 中間JSONファイルが出力されないこと。
- テキストデザインケースでは、本文テキストを欠落させず、対象装飾に対応するMarkdownまたはHTMLが出力されること。
- 画像ケースでは、Markdown内の画像リンク数とresourceフォルダ配下の画像ファイル数が入力画像数と一致すること。
- 画像リンクは相対パスであり、`data:image/` を含まないこと。
- Markdown tableケースでは、表ごとにヘッダー区切り行 `|---` が1行以上出力されること。
- HTML tableレイアウト不使用ケースでは、Markdown内にレイアウト目的の `<table>` が出力されないこと。
- Mermaid候補ケースでは、Mermaidコードブロックと `flowchart` が出力されること。
- チャートケースでは、Mermaidコードブロックを優先し、Mermaid化できない場合はチャートデータ表が出力されること。
- 通常サイズの単一ファイルは3秒以内にMarkdown出力まで完了すること。
- エラー系では、Markdownとresourceを出力せずエラー終了すること。
- 警告系では、対象要素を除外し、変換可能な要素を含むMarkdownが出力されること。
