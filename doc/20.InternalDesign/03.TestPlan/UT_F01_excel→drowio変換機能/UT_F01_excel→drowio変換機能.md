# UT_F01 Excel→Draw.io変換機能 テスト仕様書

## 1. 目的

Excel→Draw.io変換機能について、主要な入力バリエーション、Excelの規模上限、複数シート、フォルダ一括変換、処理時間の観点で確認する。

## 2. テスト方針

- 変換対象は `.xlsx` とする。
- 出力は入力ファイルと同一フォルダのDraw.io XMLとする。
- 中間JSONファイルは出力しない。
- 画像はbase64としてDraw.io XML内に埋め込む。
- Excel表示との完全一致は判定対象外とし、本仕様書に記載したXML要素数、style値、出力件数、処理時間を合格基準とする。

## 3. テスト観点

| 観点 | 確認内容 | 期待結果 |
|---|---|---|
| テキスト | 通常文字列、数値、改行を含むセル | Draw.io上に文字列として表示される |
| テーブル | 罫線付き範囲、表形式データ | 表範囲ごとに `value` 内へ `<table` が1件以上出力される |
| カラー | セル背景色、図形塗り色、線色 | 入力色と同じRGB値が `background:#RRGGBB`、`fillColor=#RRGGBB`、`strokeColor=#RRGGBB` のいずれかで出力される |
| 太字 | 太字セル、太字テキスト | 太字対象のDraw.io要素に `fontStyle=1` またはHTML内の `font-weight:bold` が出力される |
| 文字サイズ | セルや図形内の文字サイズ | 入力文字サイズptと同じ数値、または±1pt以内の `fontSize` が出力される |
| 図・オブジェクト | Excel図形、角丸図形、矢印、コネクタ | 図形数と同数の `vertex="1"`、矢印数と同数の `edge="1"` が出力される |
| 画像 | Excelに貼り付けられた画像 | 画像数と同数の `shape=image` と `data:image/` が出力される |
| 空白セル | 値もスタイルもないセル | 空白セルに対応する `cell-<セル番地>` が出力されない |
| 複数シート | 1〜100シート | 入力シート数と同数の `<diagram` が出力され、diagram名がシート名と一致する |
| 最大行列 | 1,048,576行 × 16,384列 | 空白セル全量に対応するmxCellを生成せず、値またはスタイルを持つセルのみ出力対象になる |
| フォルダ一括 | 1〜100ファイル | 入力Excel件数と同数のDraw.io XMLが同一フォルダに出力される |
| 処理時間 | 通常サイズの単一ファイル | 3秒以内にDraw.io XML生成が完了する |

## 5. 性能目標

| 項目 | 目標 |
|---|---|
| 通常サイズの単一Excel | 3秒以内 |
| 複数シート | 100シートまで対応 |
| フォルダ一括 | 100ファイルまで対応 |
| 最大行列 | 1,048,576行 × 16,384列を仕様上の上限として考慮 |

3秒目標は通常サイズの単一ファイルを対象とする。
最大行列、100シート、100ファイル、画像多数のケースでは、3秒を超える可能性がある。

## 6. 合格基準

- 正常系の全ケースでDraw.io XMLが出力され、XML parseが成功すること。
- 画像ケースでは、入力画像数以上の `data:image/` がXML内に存在すること。
- 空白セルケースでは、空白セル全量に比例したmxCellが生成されないこと。
- 複数シートケースでは、入力シート数と `<diagram` 件数が一致すること。
- 通常サイズの単一ファイルは3秒以内にDraw.io XML出力まで完了すること。
- エラー系では、Draw.io XMLを出力せずエラー終了すること。
- 警告系では、対象要素を除外し、変換可能な要素を含むDraw.io XMLが出力されること。
